### ‚úì V4-2 ‚Äî FXP state liveness

D4's carried state (`phase`, `tmem_col`, `seq_len_b`, `b`, `cta_page_start`, `bt_cache`, Q_smem content, `mbar_addr`) is alive across all page iterations:
- `phase` is a thread-private register, never overwritten except by `phase ^= 1` after each wait. ‚úì
- `tmem_col` is loaded from alloc_slot_smem at P1 and kept in registers through P16. ‚úì
- `seq_len_b`, `b`, `cta_page_start` computed at P0, never overwritten. ‚úì
- `bt_cache[pidx]` is SMEM, written once per CTA, persists through page loop. ‚úì
- Q_smem content is written once (P4) and not overwritten inside the page loop ‚Äî MMA reads it via `a_desc`. ‚úì
- `mbar_slot` is SMEM, written once at P3, read-modified by `tcgen05.commit` and `mbarrier.try_wait` each iter. ‚úì

**PASS.**

### ‚úì V4-3 ‚Äî MMA issue warp ‚âÝ alloc warp race check

P1 (alloc, warp 1) ‚Üí `__syncthreads` ‚Üí P2, P3, P4 (also barriered). By the time P8 executes, alloc's result is visible in `alloc_slot_smem` and been read into register `tmem_col` by all threads (loaded at P1-exit broadcast). Warp 0's elected thread uses `tmem_col` in `tcgen05.mma [tmem_col], ...` ‚Äî fully synchronized. **No race. PASS.**

### ‚úì V4-4 ‚Äî mbarrier count/arrival balance

- Init: `mbarrier_init(mbar, 1)` ‚Äî expects 1 arrival per phase.
- Per-iter arrivals: exactly one `tcgen05.commit` (one elected thread in warp 0). `.mbarrier::arrive::one` sends exactly 1 arrival.
- Per-iter waits: `mbarrier_wait(mbar, phase)` ‚Äî all threads spin until the 1 arrival is registered. After success, the mbarrier auto-resets its count (per mbarrier.try_wait.parity semantics).

Arrival count = 1, expected count = 1 per phase. Balanced. **PASS.**

### ‚úì V4-5 ‚Äî Address max-checks

- Q_smem max: offset(m=63, k=127) = 3*2048 + 7*256 + 1*128 + 7*16 + 15 = 8191 (byte). Buffer size 8192 B. ‚úì
- K_smem max: same as Q ‚úì
- scale_smem max: tok=63 ‚Üí 63*4 = 252 < 256 ‚úì
- scores_smem max: (m=63, n=63) ‚Üí (63*64+63)*4 = 16380 < 16384 ‚úì
- bt_cache max: pidx=3 ‚Üí 3*4 = 12 < 16 ‚úì
- alloc_slot_smem: 4 B, single slot ‚úì
- mbar_slot: 8 B, single slot ‚úì
- scores_all GMEM max: `b*64*N_max + m*N_max + (max_np-1)*64 + 63` ‚Äî always within allocated `[B, 64, N_max]` tensor (N_max = max_np * 64). ‚úì

**PASS.**

### ‚úì V4-6 ‚Äî Single-path dealloc

D4 shows: page loop with `break` on `page_id >= max_num_pages` falls through to dealloc. No `return` inside the loop. **PASS** (implementation must verify this literally ‚Äî V4 flags it).

### ‚úì V4-7 ‚Äî `asm volatile` memory clobbers

Required memory clobbers:
- `tcgen05.alloc`: writes alloc_slot_smem ‚Üí `: "memory"` ‚úì
- `tcgen05.commit`: updates mbar (SMEM) ‚Üí `: "memory"` ‚úì
- `tcgen05.fence::after_thread_sync`: ordering fence ‚Üí `: "memory"` ‚úì
- `tcgen05.ld`: reads TMEM ‚Üí no SMEM/GMEM effect, but `: "memory"` defensively ‚úì
- `tcgen05.wait::ld`: barrier ‚Üí `: "memory"` ‚úì
- `tcgen05.dealloc`: releases TMEM ‚Üí `: "memory"` ‚úì
- `mbarrier.init`, `mbarrier.try_wait`, `mbarrier.arrive.expect_tx`: SMEM effects ‚Üí `: "memory"` ‚úì

Implementation must ensure each asm block has the clobber. **PASS (instruction).**

### ‚úì V4-8 ‚Äî Phase XOR placement

D4 sequence: `mbarrier_wait(mbar_addr, phase); phase ^= 1;` ‚Äî XOR AFTER wait, BEFORE next iter's commit. Consistent with mbarrier parity semantics (even phases use 0, odd phases use 1, flipping after each satisfied wait). ‚úì

**PASS.**

### ‚úì V4-9 ‚Äî GT-17 compliance

P12 (tcgen05.ld) issues on all 32 warp lanes ‚Äî D4 explicitly: "all 32 lanes per warp (GT-17)". P14 (scale apply + write scores_smem) is gated by `if (warp_lane < 16)` ‚Äî AFTER load, not before. ‚úì

**PASS.**

### ‚úì V4-10 ‚Äî Unroll hints

- Slab loop (4 iters inside P8): NO unroll (GT-14). ‚úì
- Page loop (PAGES_PER_CTA iters wrapping P5-P15): compiler's choice; D4 doesn't mandate. Safe either way ‚Äî page iterations are thread-uniform and don't touch MMA ordering. Will not apply `#pragma unroll` to the page loop in implementation (leaves it a true dynamic loop ‚Äî safer and smaller code).

**PASS.**

- Final `scores_all[b, h, t]` = `scale[t] * Œ£_k Q_fp32[b,h,k] * fp8_to_fp32(K_raw[t,k])` = `Œ£_k Q_fp32[b,h,k] * (fp8_to_fp32(K_raw[t,k]) * scale[t])` = `Œ£_k Q_fp32[b,h,k] * K_dequant[t,k]` = Tier 2 `torch::bmm(q_fp32, K_deq.T)[b,h,t]`. ‚úì Bit-equivalent up to FP32 summation order (which the dataset test tolerance allows per GT-BASELINE's 0.00 rel_err achievement).

- Wrong scores ~50-80% of reference ‚Üí SBO/LBO descriptor mismatch (shouldn't happen; GT-11 values confirmed, but possible if make_smem_desc bit-field composition is wrong).
