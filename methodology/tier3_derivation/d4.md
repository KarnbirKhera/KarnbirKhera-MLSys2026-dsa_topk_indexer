## Decision: Warp assignment (addresses V3 Note 2)

**BOUND per-warp roles for the new `fp8_mma_scores_kernel`:**

| Warp | warp_id | Role |
|---|---|---|
| Warp 0 | 0 | Elects one thread (via `elect_sync()`) to issue: (a) mbarrier_init; (b) all `tcgen05.mma` slab issues; (c) `tcgen05.commit`. Participates in cooperative Q/K/scale loads and output write. Owns TMEM lanes 0-31 (live M-rows 0..15) for tcgen05.ld. |
| Warp 1 | 1 | `tcgen05.alloc` (kernel entry) and `tcgen05.dealloc` (kernel exit). Participates in cooperative loads and output write. Owns TMEM lanes 32-63 (live M-rows 16..31). |
| Warp 2 | 2 | Owns TMEM lanes 64-95 (live M-rows 32..47). Participates in cooperative loads and output write. |
| Warp 3 | 3 | Owns TMEM lanes 96-127 (live M-rows 48..63). Participates in cooperative loads and output write. |

Rationale: GT-2 mandates `tcgen05.alloc` by warp 1; no analog constraint on the MMA-issuing warp. Using warp 0 for MMA issue keeps warp-0 and warp-1 concerns separate and matches the gau-nernst pattern.

## Decision: mbarrier and phase tracking (addresses V3 Note 1)

**BOUND:** one SMEM mbarrier at a fixed offset. Initialized once at kernel entry (by warp 0 thread 0). Reused across all PAGES_PER_CTA page iterations via phase-XOR.

```
at kernel entry (before any page iter):
    if (warp_id == 0 && threadIdx.x == 0) mbarrier_init(mbar_addr, 1);
    __syncthreads();
    int phase = 0;

per page iteration:
    ‚Ä¶ (cooperative loads, __syncthreads) ‚Ä¶
    elected thread in warp 0:
        for (s = 0..3)  tcgen05.mma(..., enable_input_d = (s != 0));
        tcgen05.commit.cta_group::1.mbarrier::arrive::one.shared::cluster.b64 [mbar_addr];
    mbarrier_wait(mbar_addr, phase);
    phase ^= 1;
    asm volatile("tcgen05.fence::after_thread_sync;" ::: "memory");
    tcgen05.ld.sync.aligned.32x32b.x64.b32 {regs‚Ä¶}, [taddr_for_warp];
    asm volatile("tcgen05.wait::ld.sync.aligned;" ::: "memory");
    ‚Ä¶ (scale, store) ‚Ä¶
```

The mbarrier count = 1 because only the elected thread in warp 0 signals it via `tcgen05.commit`. All threads (via `mbarrier_wait`) spin until that single arrival completes. Per PTX ISA ¬ß9.7.16.12, the `.mbarrier::arrive::one` qualifier causes exactly one arrive per commit, matching the init count of 1.

Phase flips after each wait ‚Äî classic double-buffer-style parity check (`mbarrier.try_wait.parity` alternates between the two phase values).

## FSM phase sequence (file 11)

**Kernel entry (before page loop):**

| Phase | Atoms | Who | What | Barrier after? |
|---|---|---|---|---|
| P0 ‚Äî kernel-scope setup | ‚Äî | all | Compute `b = blockIdx.y`, `cta_page_start = blockIdx.x * PAGES_PER_CTA`, load `seq_len_b = seq_lens[b]`, `max_np = max_num_pages` | NO (thread-private regs) |
| P1 ‚Äî TMEM alloc | FXP, REL | warp 1 | `tcgen05.alloc [alloc_slot_smem], 64` | YES (`__syncthreads`) ‚Äî alloc_slot_smem visible to all |
| P2 ‚Äî block_table cache | AFM | warp 0 thread 0 | Load `block_table[b, cta_page_start + 0..3]` into `bt_cache[4]` SMEM; zero for out-of-range | YES (`__syncthreads`) |
| P3 ‚Äî mbarrier init | REL | warp 0 thread 0 | `mbarrier_init(mbar_addr, 1)`; `phase = 0` | YES (`__syncthreads`) ‚Äî mbar visible |
| P4 ‚Äî Q cooperative load | AFM, MOR, SYM | all 128 threads | Load Q [64,128] FP8 GMEM ‚Üí Q_smem (8√óT layout). 4 uint4 per thread. | YES (`__syncthreads`) ‚Äî Q_smem ready for MMA |

**Per-page loop (PAGES_PER_CTA iterations):**

| Phase | Atoms | Who | What | Barrier after? |
|---|---|---|---|---|
| P5 ‚Äî page guard | PRD | all | `page_id = cta_page_start + pidx`; if `page_id >= max_num_pages` ‚Üí break | NO |
| P6 ‚Äî K cooperative load w/ zero-fill | AFM, PRD, MOR | all | For each of 4 uint4 per thread: compute `global_tok = page_id*64 + tok`; if `global_tok >= seq_len_b` write zero uint4 to K_smem (8√óT layout); else load from `k_cache + bt_cache[pidx] * 8448 + tok*128 + k_base` | NO (scale load follows) |
| P7 ‚Äî scale cooperative load w/ zero-fill | AFM, PRD | threads 0..63 | If `global_tok >= seq_len_b` write 0.0 to `scale_smem[tok]`; else load `*(float*)(k_cache + phys*8448 + 8192 + tok*4)` | YES (`__syncthreads`) ‚Äî K_smem and scale_smem ready |
| P8 ‚Äî MMA issue (4 slabs) | SRG, FXP | warp 0 elected | 4 `tcgen05.mma.cta_group::1.kind::f8f6f4` (slab 0 enable_d=false, 1..3 true). **NO #pragma unroll** (GT-14). | NO (commit follows) |
| P9 ‚Äî commit | REL | warp 0 elected | `tcgen05.commit.cta_group::1.mbarrier::arrive::one.shared::cluster.b64 [mbar_addr]` (GT-18 form) | NO |
| P10 ‚Äî mbarrier wait | REL | all | `mbarrier_wait(mbar_addr, phase); phase ^= 1;` | NO (wait provides ordering) |
| P11 ‚Äî fence::after_thread_sync | REL | all | `asm volatile("tcgen05.fence::after_thread_sync;" ::: "memory")` (GT-2 step 5) | NO |
| P12 ‚Äî tcgen05.ld | AFM, SRG | all 32 lanes per warp (GT-17) | `tcgen05.ld.sync.aligned.32x32b.x64.b32 {r0..r63}, [(warp_id*32)<<16 | tmem_col]` | NO (wait::ld follows) |
| P13 ‚Äî tcgen05.wait::ld | REL | all | `asm volatile("tcgen05.wait::ld.sync.aligned;" ::: "memory")` | NO |
| P14 ‚Äî scale apply + masked write to scores_smem | AFM, PRD, SYM | all (masked by `lane<16`) | For lane L<16 in warp W: for col c in [0,64): `scores_smem[(W*16+L)*64 + c] = r[c] * scale_smem[c]`. For L‚â•16: skip (GT-17 post-load mask). | YES (`__syncthreads`) ‚Äî scores_smem ready for GMEM store |
| P15 ‚Äî cooperative GMEM store | AFM, SYM | all 128 threads | For i in [0, 128): flat = i*128 + threadIdx.x; m = flat/64; n = flat%64; `scores_all[b*64*N_max + m*N_max + page_id*64 + n] = scores_smem[m*64 + n]` | YES (`__syncthreads`) ‚Äî scores_smem reusable next iteration |

**Kernel exit (after page loop):**

| Phase | Atoms | Who | What | Barrier after? |
|---|---|---|---|---|
| P16 ‚Äî TMEM dealloc | ‚Äî | warp 1 | `tcgen05.dealloc.cta_group::1.sync.aligned.b32 tmem_col, 64` (single exit path) | NO (kernel exits) |

**FSM total barriers (`__syncthreads`):** 5 at entry (P1, P2, P3, P4 ‚Äî 4 barriers; plus P14/P15 barriers) + 2 per page iter (P7‚ÜíP8 and P14‚ÜíP15, plus P15‚Üínext-P6) = 5 + 3 √ó PAGES_PER_CTA = 5 + 12 = 17. mbarrier waits are separate.

**FXP state carried across page iterations:** `phase` (int, 1 bit used), `tmem_col` (int, allocated once), `seq_len_b`, `b`, `cta_page_start`, `bt_cache[4]`, Q_smem (refer via SMEM addr ‚Äî content constant), mbar_addr.

**No REDUCE_MERGE phase** ‚Äî no Combine Group (single REDUCE dim D). ‚úì V2 confirmed.

**No GATE phase inside this kernel** ‚Äî top-K runs in a separate kernel launch. ‚úì D1 confirmed.

## Lifetime tables (file 12)

### Block‚ÜíShared (SMEM) buffer lifetimes

| Buffer | Alive from | Alive to | Reuse count | Justification |
|---|---|---|---|---|
| Q_smem (8 KiB) | P4 (load) | kernel exit | PAGES_PER_CTA (= 4) | Q constant per CTA; reused across page loop. Reuse > 1 ‚úì (file 12 MEA row). |
| K_smem (8 KiB) | P6 (load) | P7 ‚Üí P14 (consumed by MMA in P8, unused after P13 per iter) | 1 per page iter | SMEM justified by MMA requiring B-operand in SMEM (Table 42 mandate). |
| scale_smem (256 B) | P7 (load) | P14 (multiply) | 16 (64 cells √ó 16 live-row threads reading) | Reuse > 1 ‚úì. |
| scores_smem (16 KiB) | P14 (write) | P15 (read) | 1 per page iter | Staging buffer for coalesced GMEM write; reuse = 1 but justified by coalescing (file 12 MEA row: coalescing is a valid alternative justification). |

| alloc_slot_smem (4 B) | P1 | P16 | persistent | Holds TMEM column base. |
| mbar_slot (8 B) | P3 | kernel exit | PAGES_PER_CTA | Reused across page iters via phase flip. |

## Address composition (file 13)

### Global addresses

- **Q**: `q_base + b * 64 * 128 + m * 128 + k_base` (FP8, byte stride 128). k_base multiple of 16 ‚Üí uint4 load.
- **K FP8**: `k_cache + phys * 8448 + tok * 128 + k_base` (FP8). k_base multiple of 16 ‚Üí uint4 load.
- **K scale**: `k_cache + phys * 8448 + 8192 + tok * 4` (FP32, byte stride 4). Single FP32 load per thread.
- **block_table**: `bt + b * max_num_pages + page_id` (i32).
- **scores_all**: `scores_all + b * 64 * N_max + m * N_max + global_n` (FP32).
- **seq_lens**: `sl + b` (i32).

All addresses are affine in their coordinate inputs. No MOR composition across levels needed (each leaf stands alone).

### SMEM addresses (after `__cvta_generic_to_shared`)

- **Q_smem**: base `Q_SMEM_OFFSET`. `smem_offset(m, k) = (k/32)*2048 + (m/8)*256 + ((k%32)/16)*128 + (m%8)*16 + (k%16)` (GT-11 formula).
- **K_smem**: base `K_SMEM_OFFSET`. Same formula as Q.
- **scale_smem**: base `SCALE_SMEM_OFFSET`. Linear: `smem_offset(tok) = tok * 4`.
- **scores_smem**: base `SCORES_SMEM_OFFSET`. Linear: `smem_offset(m, n) = (m * 64 + n) * 4`.
- **bt_cache**: base `BT_CACHE_OFFSET`. Linear: `smem_offset(p) = p * 4`.
- **alloc_slot_smem**: base `ALLOC_SLOT_OFFSET`. Single 4-B slot.
- **mbar_slot**: base `MBAR_SLOT_OFFSET`. 8-B aligned.

**SMEM layout (all offsets 16-B aligned for descriptor compliance):**

```
0             : Q_smem (8192 B)
8192          : K_smem (8192 B)
16384         : scores_smem (16384 B)
32768         : scale_smem (256 B)
33024         : bt_cache (16 B)
33040         : alloc_slot_smem (4 B, round up to 16 B)
33056         : mbar_slot (8 B, round up to 16 B)
Total: 33072 B
```

### TMEM addresses

**taddr for warp W's tcgen05.ld**: `(W * 32) << 16 | tmem_col`
- W=0: `tmem_col | 0x00000000`
- W=1: `tmem_col | 0x00200000`
- W=2: `tmem_col | 0x00400000`
- W=3: `tmem_col | 0x00600000`

`tmem_col` is the 16-bit column base returned by `tcgen05.alloc` (read from `alloc_slot_smem`).

### SMEM descriptor helpers

```
desc_encode(x) = (x & 0x3FFFF) >> 4    // 14-bit field, units of 16 B

make_smem_desc(smem_addr_shared):
    d = desc_encode(smem_addr_shared)        // bits 0-13
    d |= desc_encode(128) << 16              // bits 16-29 : LBO encoded = 8
    d |= desc_encode(256) << 32              // bits 32-45 : SBO encoded = 16
    d |= 0b001 << 46                         // bits 46-48
    return d                                 // bits 49+ all 0
```

**Per-MMA-issue descriptors:**
```
a_desc = make_smem_desc(Q_SMEM_OFFSET + slab * 2048)
b_desc = make_smem_desc(K_SMEM_OFFSET + slab * 2048)
```
(same shared-addr base used for all 4 slabs in this kernel since Q and K are both in SMEM at fixed offsets; only the slab-relative 2048-B step changes)

### Thread‚Üílocal address (functorial substitution from block‚Üíthread)

From D3 Decision 6 (Q loader), the per-thread per-iter affine map:
```
u(i) = threadIdx.x * 4 + i         i ‚àà [0, 4)
m = u / 8
k_uint4 = u % 8
k_base = k_uint4 * 16
```

Global Q addr per thread per iter: `q_base + b*8192 + m*128 + k_base`.
SMEM dst addr per thread per iter: `Q_SMEM_OFFSET + (k_base/32)*2048 + (m/8)*256 + ((k_base%32)/16)*128 + (m%8)*16` (k_base%16 = 0 since multiple of 16).

K loader uses the same formula substituting `tok` for `m` and adding `phys * 8448 + tok*128` for the global-side base.

## Single-path TMEM dealloc (addresses V3 Note 3)

The kernel has **exactly one dealloc site** at kernel exit. The page loop may break early when `page_id >= max_num_pages`, but the break path falls through to the dealloc block, not to `return`. Concretely:

```cpp
// page loop
for (int pidx = 0; pidx < PAGES_PER_CTA; ++pidx) {
    int page_id = cta_page_start + pidx;
    if (page_id >= max_num_pages) break;     // falls out of the loop, not kernel
    ‚Ä¶ P6 ‚Ä¶ P15 ‚Ä¶
}
// single-path dealloc
if (warp_id == 1) tcgen05_dealloc(tmem_col, 64);
```

No early `return` statements inside the page loop.

## What V4 must verify (Opus audit)

1. FSM barrier placement: every write‚Üíread cross-thread relation has a `__syncthreads` between writer phase and reader phase.
2. FXP state variables (phase, tmem_col, seq_len_b, etc.) are alive in every phase that reads them.
3. MMA issue warp (warp 0) does not race with alloc warp (warp 1) ‚Äî alloc completes + syncthreads before any MMA issue.
4. mbarrier init (count=1) matches commit arrivals (exactly 1 per commit).
5. Address max-checks pass for every SMEM buffer.
6. Single-path TMEM dealloc ‚Äî no early `return` below alloc.
7. All `asm volatile` fences carry `: "memory"` clobber where needed (commit, fence::after, ld, wait::ld).
8. Loop-carried phase flip ‚Äî confirm the phase XOR appears after every `mbarrier_wait`, before the next iter's `tcgen05.commit`.
9. GT-17 compliance in P12/P14: all 32 warp lanes issue tcgen05.ld; masking happens in P14 (not P12).
10. No `#pragma unroll` on the slab loop (GT-14); `#pragma unroll 4` on the page loop IS acceptable (page loop is a thread-uniform counted loop and doesn't affect MMA ordering ‚Äî but to stay conservative, D4 recommends leaving the page loop unrolled either way, the compiler decides).
11. No early `return` anywhere below `tcgen05.alloc`.
12. SMEM alignment: Q_smem, K_smem, scores_smem, scale_smem, bt_cache, alloc_slot_smem, mbar_slot all 16-B aligned.
