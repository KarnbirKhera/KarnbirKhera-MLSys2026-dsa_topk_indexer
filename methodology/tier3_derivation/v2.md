### ✓ Audit item 3 — SRG vs MON classification for D-reduction

File 03 SRG row: sum over k is ASC + IDE + COM + DIS. All four properties hold for FP32 accumulation of FP8×FP8 products:

- IDE: identity = 0.
- COM: float addition is commutative.
- DIS: multiplication distributes over addition.

Classification SRG is confirmed. This authorizes tensor-core MMA as the warp-level implementation per file 03 SRG GPU manifestation: "tensor core MMA for dot products."

### ✓ Audit item 4 — 4-slab accumulation is SRG × FXP (file 03 Matrix11×5 SRG row, "Online merge" cell)

File 03: "Online merge: merge(acc, new_partial). Fixed-point tile loop: accumulate partial semiring each iteration. Online softmax is this pattern."

For plain sum semiring, the online merge is just `acc = acc + new_partial`. Hardware implements this by setting `enable_input_d=true` from slab 1 onward — TMEM is the register-carry state. Slab 0 uses `enable_input_d=false` (fresh accumulator, identity = 0). This matches file 03's formulation.

**NOT** a Combine Group — D2 is correct that Combine Groups are inter-dimension relations (file 09 REL row: "two REDUCE dimensions whose outputs feed the same output value"), and the 4 slabs are partitions of one dimension D=128.

## Notes forwarded to D3

1. **Tile shape candidates:** MMA_N=64 or MMA_N=128. D3 must pick one, citing occupancy and page-straddling behavior. PAGES_PER_CTA for K gather — GT-13 minimum = 2 — matters only if we ever do a multi-CTA-per-page pattern. For this kernel, each CTA owns its own N-tile; GT-13 constraint does not directly apply (single-CTA-per-N-tile). **Confirm in D3.**
2. **IDESC and SMEM descriptor encoding** — exact bit fields from PTX ISA Tables 42 (SMEM desc) and 44 (IDESC). SBO/LBO values for K-major 8×T confirmed by GT-11: SBO field = 16 (→ 256 B), LBO field = 8 (→ 128 B). transpose_B bit = 0 (GT-15).
3. **Q loading decision:** cooperative vs TMA. Small Q tile (8 KiB) + single use per N-tile iteration → cooperative preferred (avoids tensor-map construction). Decide in D3.
4. **Slab loop: NO unroll** (GT-14). Already noted.
5. **Post-MMA path:** TMEM → register → scale multiply (per-column) → SMEM staging → GMEM (scores_all[b, h, t]). Detail in D3/D4.
6. **Padding zero-fill:** K SMEM tile rows for `global_tok ≥ seq_lens[b]` must be zeroed before MMA. This must be done by the cooperative loader path.
7. **Kernel launch:** new custom kernel replaces both `dequant_gather_batched_kernel` and `torch::bmm`. Grid = `(ceil(N_max / MMA_N), B)`. Block threads = 4 warps × 32 = 128.
