### ✓ Audit item 2 — Streaming Gate detection rule executed

File 08, section "How to detect it":
> "Are the GATE dimension's candidates produced inside the FXP tile loop, or do they all exist before the loop starts? If inside, you have a Streaming Gate."

D1 answered:
1. Delta's FXP loop is over K=128 (4 slabs of K=32). The K loop computes the inner product for a single N candidate — it is not a loop over N candidates.
2. The implicit FXP over N across CTAs materializes all N candidates in `scores_all` before `at::topk` runs (candidates visible at the kernel-launch boundary).
3. Outcome: single-pass Gate.

File 08 also lists six pre-wired decisions required when Streaming Gate IS triggered (accumulation buffer, staging scratch, two-case merge, staging sort, buffer re-sort, deferred output write). None of these apply to this delta, which is correct given the detection result.

**PASS.**

### ✓ Audit item 4 — Correctness identity has no hidden sign assumption

D1 originally proposed a scale-deferral identity relying on `relu(x*s) = s*relu(x)` for `s ≥ 0`. The Python reference at
`mlsys26-contest/definitions/dsa_paged/dsa_topk_indexer_fp8_h64_d128_topk2048_ps64.json` (run()) does **not** assert non-negative scales, so an adversarial or corrupted workload could in principle contain a negative scale and violate the identity.

D1 has been corrected to use the plain distributive law `Σ_k Q*(fp8_to_fp32(K)*s) = s*Σ_k Q*fp8_to_fp32(K)` and apply the scale **pre-relu, post-MMA** — the scale is multiplied into the TMEM output before relu is applied downstream. This holds for any real `scale_tok`. `scores_all` becomes bit-equivalent to Tier 2's `torch::bmm(q_fp32, K_deq.T)` up to FP8→FP32 rounding order.

**PASS.**

### ✓ Audit item 5 — Padding-mask carry-over from GT-26

GT-26 established that padding rows (tok ≥ seq_len) in K_deq are zero-filled so bmm produces 0 at those columns, which are then stamped to `-INF` by the scoring kernel. D1 preserves this exact mechanism: the cooperative K SMEM fill in the new kernel zero-fills FP8 K bytes for tok ≥ seq_len rows, FP8 zero bytes decode to 0.0, MMA produces 0, and the unchanged scoring kernel stamps -INF. No deviation from GT-26 required.

**PASS.**
