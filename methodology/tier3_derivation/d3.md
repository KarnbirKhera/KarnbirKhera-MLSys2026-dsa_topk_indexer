## Scope continuity

D3 binds the D2 structural analysis to specific B200 sm_100a hardware primitives: MMA tile shape, IDESC bits, SMEM descriptor bits, grid/block topology, SMEM layout formulas, TMEM lifecycle. All decisions are COMMITTED (Section 15). No "could/optional".

## Decision 1 — MMA kind, shape, accumulator

**BOUND: `tcgen05.mma.cta_group::1.kind::f8f6f4`** with dense matrices, no `.ws`, no sparsity.

**BOUND tile shape: M=64, N=64, K=32 (per slab).**

Justification from Table 41 (`tcgen05_mma_shapes.txt`, `.kind::f8f6f4`, No `.ws`, CTA Group 1, Dense):
- Row permits `64xNxK` with N ∈ {8, 16, ..., 256} steps of 8, K fixed at 32.
- H (num query heads) = 64 matches M=64 exactly — **one MMA issue covers all heads for one N-tile** (no M-tiling).
- N=64 covers exactly one FP8 K-cache page (64 tokens) — **no page-straddling per MMA issue**, simplifying the cooperative K loader.
- K=32 is mandatory per Table 41 footnote (§9.7.16.2.2): "if K can be uniquely determined as per the Table 41, then K cannot be explicitly specified."

**REQUIRED slab count:** D=128 / K=32 = **4 slabs**.

**BOUND accumulator:** F32 (dtype bits 4-5 = 1 per Table 44).

**BOUND input types:** both A (Q) and B (K) as E4M3 (FP8 e4m3fn per Python reference). atype = E4M3 = 0, btype = E4M3 = 0 per Table 44.

## Decision 2 — PAGES_PER_CTA, grid and block topology

**BOUND PAGES_PER_CTA = 4.** Rationale:
- Amortizes Q load (8 KiB per batch) across 4 pages — cuts redundant Q HBM traffic ~4× vs PAGES_PER_CTA=1.
- Respects GT-13 minimum (≥ 2) even though GT-13's non-determinism was observed in a different multi-CTA-scratch context; applying the floor here is defensive and cheap.
- SMEM budget (≤ 33 KiB) permits multiple CTAs per SM.

**BOUND grid:**
```
grid.x = ceil(max_num_pages / PAGES_PER_CTA)
grid.y = B (batch_size)
```

**BOUND block:** 128 threads = **4 warps × 32 lanes**.
- GT-12: M=64 distributed 16 rows/warp → 4 warps total carry all M rows for tcgen05.ld.
- Warp 1 issues `tcgen05.alloc` (GT-2).

**BOUND per-CTA page iteration:** a CTA's page index range is
```
cta_page_start = blockIdx.x * PAGES_PER_CTA
for page_idx_in_cta in [0, PAGES_PER_CTA):
    page_id = cta_page_start + page_idx_in_cta
    if page_id >= max_num_pages: break
    … (load, MMA, store) …
```

## Decision 3 — SMEM layout (GT-11 8×T core-tile; GT-10 swizzle = NONE)

**BOUND SMEM organization per CTA:**

| Buffer | Bytes | Layout | Notes |
|---|---|---|---|
| Q_smem | 8192 | 8×T core-tile, K-major, 4 K-slabs of 2048 B each | loaded ONCE at CTA entry, reused PAGES_PER_CTA times |
| K_smem | 8192 | 8×T core-tile, K-major, 4 K-slabs of 2048 B each | reloaded per page |
| scale_smem | 256 | linear `[64] FP32` | reloaded per page |
| scores_smem | 16384 | linear `[64 rows × 64 cols] FP32` | reused per page; contiguous along N |
| block_table_cache | 16 | linear `[4] int32` | loaded once at CTA entry |
| mbarrier | 16 | u64, 16-B aligned | one mbarrier for MMA-completion signalling |
| **Total** | **~33 KiB** | | fits ≥ 6 CTAs per SM on B200 SMEM budget |

**Dynamic SMEM opt-in required:** 33 KiB < 48 KiB → actually NOT required, but we'll request 36 KiB explicitly via `cudaFuncSetAttribute(kernel, cudaFuncAttributeMaxDynamicSharedMemorySize, …)` anyway to be future-proof if we raise PAGES_PER_CTA later. Keep it at static SMEM for now: extern `__shared__` declaration.

**BOUND 8×T fill formula (GT-11 confirmed):**
For an element at row `m ∈ [0, M_tile)`, column `k ∈ [0, K_tile)` (K_tile = 128 for Q and K since the full D is staged):
```
smem_offset(m, k) = (k / 32) * SLAB_BYTES
                  + (m / 8) * SBO_BYTES
                  + ((k % 32) / 16) * LBO_BYTES
                  + (m % 8) * 16
                  + (k % 16)
```
where `SLAB_BYTES = 2048`, `SBO_BYTES = 256`, `LBO_BYTES = 128`.

Per-slab SMEM descriptor base address = `Q_smem_base + slab * 2048` (same for K_smem).

**BOUND swizzle = NONE (GT-10).** The 8×T core-tile is linear (no hardware swizzle); the MMA SMEM descriptor's swizzle field (bits 61-63) must be 0.

## Decision 4 — IDESC bits (Table 44, kind::f8f6f4)

**BOUND IDESC value = `0x04100010` (uint32).** Bit-by-bit (Table 44):

| Bits | Field | Value |
|---|---|---|
| 0-1 | Sparsity selector | 0 (dense) |
| 2 | Sparsity | 0 (Dense) |
| 3 | Saturate | 0 (N/A for FP types) |
| 4-5 | dtype (D) | **F32 = 1** → bit 4 set |
| 6 | Reserved | 0 |
| 7-9 | atype (A) | **E4M3 = 0** |
| 10-12 | btype (B) | **E4M3 = 0** |
| 13 | Negate A | 0 |
| 14 | Negate B | 0 |
| 15 | Transpose A | 0 (K-major, GT-15 analog) |
| 16 | Transpose B | 0 (GT-15 direct: transpose_B=1 produces 50-80% wrong-magnitude descriptor mismatch signature) |
| 17-22 | N >> 3 | **64 >> 3 = 8** → bit 20 set (0x100000) |
| 23 | Reserved | 0 |
| 24-28 | M >> 4 | **64 >> 4 = 4** → bit 26 set (0x04000000) |
| 29 | Reserved | 0 |
| 30-31 | `.ws` max shift | 0 (not using `.ws`) |

Composition: `(1 << 4) | (8 << 17) | (4 << 24) = 0x10 | 0x100000 | 0x04000000 = 0x04100010`. ✓

## Decision 5 — SMEM descriptor bits (Table 42, GT-11 values)

**BOUND helper:**
```cpp
__device__ __forceinline__ uint64_t desc_encode(uint64_t x) {
    return (x & 0x3FFFFULL) >> 4ULL;
}

__device__ __forceinline__ uint64_t make_smem_desc(uint32_t smem_addr_shared) {
    uint64_t d = 0;
    d |= desc_encode((uint64_t)smem_addr_shared);      // bits 0-13 matrix start
    // bits 14-15 reserved = 0
    d |= desc_encode((uint64_t)128ULL) << 16;          // bits 16-29 LBO encoded = 8 (LBO=128 B per GT-11)
    // bits 30-31 reserved = 0
    d |= desc_encode((uint64_t)256ULL) << 32;          // bits 32-45 SBO encoded = 16 (SBO=256 B per GT-11)
    d |= (uint64_t)0b001ULL << 46;                     // bits 46-48 fixed constant
    // bits 49-51 base offset = 0 (SWIZZLE_NONE, pattern starts at 0)
    // bit 52 stride mode = 0 (byte offset relative)
    // bits 53-60 reserved in practice = 0
    // bits 61-63 swizzle = 0 (No swizzling, GT-10)
    return d;
}
```

**Per-slab descriptors (constructed once per MMA issue):**
```
a_desc = make_smem_desc(Q_smem_generic_to_shared + slab * 2048)
b_desc = make_smem_desc(K_smem_generic_to_shared + slab * 2048)
```

where `_generic_to_shared = __cvta_generic_to_shared(ptr)`.

**Constraint check (Table 42):** Matrix start, LBO byte offset, and SBO byte offset must be 16-B-aligned.
- Matrix start: SMEM arrays are 16-B aligned via `__align__(16)` on the `extern __shared__` declaration. ✓
- LBO = 128 B: multiple of 16. ✓
- SBO = 256 B: multiple of 16. ✓

## Decision 6 — Q loader (cooperative, not TMA)

**BOUND cooperative Q load (once at CTA entry):**

Q global layout: `[B, 64, 128]` FP8, row-major. Per-batch Q slab = 8192 B.

- 128 threads × 4 uint4 per thread = 512 uint4 = 8192 B. ✓
- Loop `i ∈ [0, 4)`; uint4 index `u = threadIdx.x * 4 + i`.
- `m = u / 8` (∈ [0, 64)), `k_uint4 = u % 8` (∈ [0, 8)), `k_base = k_uint4 * 16`.
- Global src: `Q_gmem + b * 64 * 128 + m * 128 + k_base` (16 B aligned — k_base is multiple of 16).
- SMEM dst: `Q_smem + (k_base/32)*2048 + (m/8)*256 + ((k_base%32)/16)*128 + (m%8)*16`.
- Load 16 bytes via `*(uint4*)dst = *(uint4*)src;`.

TMA not used for Q despite being permitted (stride 128 B is a multiple of 16). Rationale: (1) single-use Q tile per CTA makes TMA setup overhead unjustified; (2) avoiding tensor-map-replace state machine simplifies host path and eliminates GT-6 risks; (3) cooperative 128-thread load saturates HBM for a single 8 KiB tensor just as well.

## Decision 7 — K loader (cooperative, TMA forbidden by GT-9, zero-fill mask)

**Per-page K load (once per page iteration inside the CTA loop):**

- 128 threads × 4 uint4 = 512 uint4 = 8192 B (= 64 tokens × 128 FP8 bytes per page). ✓
- Loop `i ∈ [0, 4)`; `u = threadIdx.x * 4 + i`; `tok = u / 8`; `k_uint4 = u % 8`; `k_base = k_uint4 * 16`.
- `global_tok = page_id * 64 + tok`.
- **Zero-fill predicate:** if `(page_id >= max_num_pages) || (global_tok >= seq_lens[b])`, write `make_uint4(0,0,0,0)` to K_smem.
- Else: `phys = block_table_cache[page_idx_in_cta]`; `src = k_cache + phys * 8448 + tok * 128 + k_base` (16 B aligned).
- SMEM dst: same 8×T formula as Q, substituting (m→tok, k→k_base).

**Scale gather (separate pass, same page):**
- 64 FP32 scales per page. Threads 0..63 each load one FP32:
- `tok = threadIdx.x`; `global_tok = page_id * 64 + tok`.
- If `page_id >= max_num_pages || global_tok >= seq_lens[b]`: write 0.0 to `scale_smem[tok]`.
- Else: `src = k_cache + phys * 8448 + 8192 + tok * 4`; `scale_smem[tok] = *(float*)src`.
- Threads 64..127 idle this pass.

**Block_table cache (once at CTA entry):**
- 4 int32s = 16 B. Thread 0 loads:
- `for (int i=0; i<PAGES_PER_CTA; ++i) { int page_id = cta_page_start + i; if (page_id < max_num_pages) block_table_cache[i] = block_table[b * max_num_pages + page_id]; else block_table_cache[i] = 0; }`
- Followed by `__syncthreads()`.

## Decision 8 — TMEM lifecycle (GT-2 canonical, adapted per-page)

**BOUND TMEM allocation:**
- N columns allocated = 64 (matches MMA_N).
- Warp 1 (threads 32..63) issues `tcgen05.alloc.cta_group::1.sync.aligned.shared::cta.b32 [alloc_slot], 64` at CTA entry.
- `alloc_slot` is a SMEM slot that receives the TMEM column base address (standard PTX pattern).
- After alloc: `__syncthreads()` so all threads see the TMEM base.

**BOUND per-page MMA sequence (no `#pragma unroll` on the 4-slab loop, GT-14):**

For each page iteration:
```
// (load K, scales — see Decision 7)
__syncthreads();

if (warp_id == 0 && elect_sync()) {
    // 4-slab MMA — slab 0 with enable_input_d=false, rest true
    // NO #pragma unroll (GT-14)
    for (int s = 0; s < 4; ++s) {
        uint64_t a_desc = make_smem_desc(Q_smem_addr + s * 2048);
        uint64_t b_desc = make_smem_desc(K_smem_addr + s * 2048);
        int enable_d = (s != 0);
        asm volatile(
            "{ .reg .pred p;\n"
            "  setp.ne.b32 p, %4, 0;\n"
            "  tcgen05.mma.cta_group::1.kind::f8f6f4 [%0], %1, %2, %3, p;\n"
            "}"
            :: "r"(tmem_base), "l"(a_desc), "l"(b_desc), "r"(IDESC),
               "r"(enable_d));
    }
    // GT-18 commit form
    asm volatile("tcgen05.commit.cta_group::1.mbarrier::arrive::one.shared::cluster.b64 [%0];"
                 :: "r"(mbar_addr) : "memory");
}

// ALL threads wait
mbarrier_wait(mbar_addr, phase);
phase ^= 1;

// GT-2 step 5: consumer-side fence after mbarrier
asm volatile("tcgen05.fence::after_thread_sync;" ::: "memory");

// GT-12 + GT-17: ALL 32 lanes issue tcgen05.ld (cannot gate by lane<16 before)
//
// M=64 distributed 16 per warp; warp w owns rows [w*16, w*16+16).
// Accumulator shape [64 rows × 64 cols]. Use shape .32x32b — one warp loads one M-row worth at a time.
// Actually for M=64/N=64 we use shape .16x256b — each warp holds 16 M-rows × 64 N-cols × 4 B = 4096 B per warp;
// 4 warps × 4 KiB = 16 KiB = scores_smem.
// tcgen05.ld.sync.aligned.16x256b.x4.b32 for 4 × 32b per lane per instruction.

uint32_t taddr_for_warp = (warp_id * 16) << 16 | lane_id;   // per GT-12 canonical
// .16x256b loads 64 FP32 cols per row, 4 registers per lane (since 256b = 8 × 32b / 2 lanes... need to re-check)

[…load TMEM values…]

// wait for ld completion
asm volatile("tcgen05.wait::ld.sync.aligned;" ::: "memory");

// Apply per-column scale (in registers) and stage to scores_smem
//    score_reg[i] = tmem_reg[i] * scale_smem[n_col[i]]

// Cooperative GMEM store:  scores_smem → scores_all[b, :, page_id*64 : page_id*64+64]
__syncthreads();
```

**BOUND tcgen05.ld shape:** **`.32x32b.x64.b32`** — one issue per warp covers the warp's full TMEM chunk.

Rationale (cross-check with `tcgen05_ld_st_wait.txt` §9.7.16.8.1 access restrictions + Table 51):
- Access restriction: warp W is permitted access only to TMEM lanes `[W*32, W*32+32)`. For M=64, 4 warps collectively cover lanes 0..127, with live data in lanes `[W*32, W*32+16)` per GT-12 (upper 16 lanes of each warp's chunk read zeros).
- `.32x32b` accesses **32 TMEM lanes per warp** — matches warp's full accessible chunk exactly.
- `.x64` repeats the 32-bit base 64 times along the column dimension → 64 × FP32 cols per TMEM-lane → covers the full N=64 accumulator width.
- Table 51 register count: `.32x32b.x64` → **64 × 32b registers per warp thread**. Per-thread mapping: warp-thread L ↔ TMEM-lane `W*32 + L`, carrying that lane's 64 FP32 column values.
- Per warp total: 32 × 64 = 2048 FP32 (includes 16 × 64 = 1024 live cells + 1024 zero-fill cells). Wasted bandwidth on the upper-16 zero cells is accepted in exchange for a clean 1-thread-per-TMEM-lane mapping.
- taddr value per warp: `(W * 32) << 16 | tmem_col_base`, where `tmem_col_base` is the column returned by `tcgen05.alloc`.

**Post-load masking (GT-17):** all 32 lanes issue `tcgen05.ld` (mandatory); the `if (warp_lane < 16)` guard lives AFTER the load, gating the register→scale→SMEM write path so only live lanes contribute.

**Register budget per thread:** 64 FP32 (TMEM result) + ~10 scalars (scale, addresses, counters) ≈ 74 reg × 4 B = under 256 B per thread — well within B200 per-thread register budget and leaves occupancy limited by SMEM (not regs).

**BOUND TMEM dealloc:** at kernel exit (single path — no early exits below alloc):
```
if (warp_id == 1) {
    asm volatile("tcgen05.dealloc.cta_group::1.sync.aligned.b32 %0, 64;" :: "r"(tmem_base));
}
```

## Decision 9 — Post-MMA scale multiply and GMEM store

**BOUND post-MMA path (per page, per warp handling its 16 M-rows):**

For each register loaded from TMEM (representing an FP32 accumulator cell at row `m = warp_id*16 + lane_id%16`, column `n ∈ warp-specific set`):
1. Read `scale_smem[n]` (each lane broadcasts the right column's scale from SMEM — SMEM read is fast).
2. `score = tmem_val * scale`.
3. Write to `scores_smem[m * 64 + n]`.
4. After all warps done: `__syncthreads()`.
5. Cooperative write `scores_smem → scores_all[b, :, page_id*64 : (page_id+1)*64]`. Output layout: `scores_all` strides = `[B, H=64, N_max]` row-major. Write 64 rows × 64 cols × 4 B = 16 KiB per page to GMEM:
   - 128 threads × 32 B/thread = 4096 B × 4 iter = 16 KiB.
   - Or: each row (64 × 4 B = 256 B) assigned to a 2-warp team — 64 rows / (128 threads / 64 cols) → one row takes 64 threads = 2 warps; 4 warps split 2 rows at a time, over 32 iters ... simpler: 128 threads × 1 float per thread × 128 iters = 16384 floats = 16 KiB. Coalesced along N.

Actual loop (simplest coalesced pattern): each thread writes one FP32 per iter; 128 threads × 128 iters = 16384 writes. Addressing: `m = iter / 2; n = (iter % 2) * 64 + threadIdx.x` — no wait that's for a different shape. Concretely:
```
for (int i = 0; i < 128; ++i) {
    int flat = i * 128 + threadIdx.x;
    int m = flat / 64;   // 0..63
    int n = flat % 64;
    scores_all[b*64*N_max + m*N_max + page_id*64 + n] = scores_smem[m*64 + n];
}
```

This is coalesced along `threadIdx.x` when `N_max` is contiguous along the `n` dim (it is — scores_all is row-major with n the fastest-moving).

Wait, actually in the line `scores_all[b*64*N_max + m*N_max + page_id*64 + n]`, with `threadIdx.x` running through `n = flat % 64`, two adjacent threads differ in `n` by 1 only if `flat % 64` increments. For flat = i*128 + tid: (tid=0, tid=1) → flat differs by 1, n differs by 1 (as long as flat%64 doesn't wrap). Across tid=0..63, flat%64 = tid (good, contiguous). For tid=64..127, flat%64 = tid-64. So within one iteration of `i`, threads split into two groups: threads 0..63 write row m = i*2, threads 64..127 write row m = i*2+1.

Both writes are stride-1 along n — coalesced within each 64-thread group. Two 128-byte transactions per iter (one per 64-thread group per row). 128 iters × 2 transactions = 256 transactions for 16 KiB. Acceptable.

**BOUND final kernel structure (after PAGES_PER_CTA loop):**
- TMEM dealloc.
- Kernel exit.
