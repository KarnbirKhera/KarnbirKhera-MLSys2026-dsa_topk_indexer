# D1 — Molecule Detection (Tier 3, scoped)

## Scope declaration (Section 10 mandate)

**(a) What this optimization changes:** Replaces the Tier 2 `dequant_gather_batched_kernel` + `torch::bmm(q_fp32_all, K_deq_all.T)` pair with a single fused tcgen05 kernel that consumes FP8 K directly (no FP32 K materialization) and writes FP32 scores to an intermediate `scores_all [B, 64, N_max]` via TMEM → register → SMEM → GMEM. The per-token FP32 scale is applied as a post-MMA **pre-relu** column multiply inside the same kernel, **before** `scores_all` is written — so `scores_all` contains fully dequantized scores bit-identical to the Tier 2 output of `torch::bmm(q_fp32, K_deq.T)` up to FP8/FP32 rounding order. This design does **not** rely on any sign assumption about `scale_tok`; relu is still applied downstream by `weighted_relu_sum_batched_kernel` on already-dequantized scores, preserving the exact Tier 2 correctness chain.

**(b) What this optimization explicitly does NOT change:**
- `weighted_relu_sum_batched_kernel` — its inputs (scores_all, weights, seq_lens) and outputs (final_scores) are bit-identical to Tier 2.
- `at::topk` on `final_scores` — same call, same K, same radix-select library path.
- `translate_indices_batched_kernel` — identical.
- GATE molecule class — top-K stays a **single-pass gate** after scores are fully materialized. **Streaming Gate is NOT introduced in this delta** (see detection check below).
- Host-side launch topology (`launch_topk_c` signature unchanged; pybind entry point unchanged; `config.toml` entry_point unchanged).
- FP8 data layout: k_cache remains `[num_pages, 64, 1, 132]` with per-token scale bytes at offset `PAGE_DATA_BYTES + tok*4` (GT identity preserved).

**(c) Checkpoint locked for revert:** `checkpoints/kernel_naive.cu` (Phase 1 baseline, 0.803 ms mean). Tier 2 state snapshotted at `tier3_derivation/kernel_tier2.cu` (0.112 ms mean) for side-by-side.

## Molecule detection (Step 0.5 → Level 0)

**Level 0 geometry (unchanged from the definition):**
- Input A: `Q_fp8 [B, H=64, D=128]` (FP8 e4m3, no per-element scale on Q).
- Input B: `K_cache_fp8` viewed as `[num_pages, P=64, D=128]` FP8 e4m3 data + per-token FP32 scale, gathered via `block_table [B, max_num_pages]` and masked by `seq_lens [B]`.
- Input C: `weights [B, H=64]` FP32, `block_table [B, max_num_pages]` int32, `seq_lens [B]` int32.
- Output: `topk_indices [B, TOPK=2048]` int32.
- Operation:  `final_scores[b,t] = Σ_h relu(Σ_k Q_fp32[b,h,k] * (fp8→fp32)(K[phys,tok,k]) * scale[phys,tok]) * weights[b,h]`, where `phys, tok = block_table[b, t/64], t%64` and `t < seq_lens[b]`; then `topk_indices[b] = argtopk_2048(final_scores[b])` padded with `-1` beyond `actual_topk = min(TOPK, seq_lens[b])`.

**Delta molecule set (what the Tier 3 change introduces or removes):**

| Molecule | Before (Tier 2) | After (Tier 3) |
|---|---|---|
| FXP (tile loop) | Implicit inside cuBLAS bmm (no explicit K-slab loop at kernel level) | **Explicit** K-slab loop of static count `ceil(D/32)=4` (kind::f8f6f4 K=32) |
| PRD (reduction over K) | cuBLAS | **Tensor-core MMA accumulation** in TMEM |
| SRG (H-reduction in scoring) | unchanged | unchanged |
| MON (top-K rank) — GATE phase | `at::topk` (radix-select) | `at::topk` (radix-select) — unchanged |
| MOR (dequant morphism) | Explicit dequant kernel writes FP32 K | **Fused into MMA** — no FP32 K materialization; scale applied post-MMA as column multiply |
| MEA | FP32 K_deq: `B*N_max*512` bytes | **Eliminated**. New intermediate: scores_all `B*64*N_max*4` bytes (already present in Tier 2, unchanged) |

**Gate Specification Table — status after Tier 3:**
- Gate Criterion: "value has rank among top-K by magnitude." Unchanged.
- Maximum Output Size: K=2048. Unchanged.
- Criterion Parallelizability: NO. Unchanged (rank-based).
- Phase 1 (criterion evaluation): radix-select via `at::topk`. Unchanged — library implementation.
- Phase 2 (index assignment): `translate_indices_batched_kernel`. Unchanged.
- REL between phases: barrier supplied by kernel-launch boundary. Unchanged.

## Streaming Gate detection (file 08 rule — mandatory check)

> "Are the GATE dimension's candidates produced inside the FXP tile loop, or do they all exist before the loop starts?"

**The GATE dimension is N (sequence positions within a batch element).**

**Tier 3 delta's FXP loop is over K (head-dim=128, 4 slabs of 32).** The K loop is nested INSIDE the computation of each individual N candidate's score — one N column needs all 4 K slabs summed before its score is known. The K loop is not a loop over GATE candidates; it is a loop inside a single GATE candidate's score computation.

**There is also an implicit FXP over N**, because the kernel grid tiles N across CTAs (each CTA produces a contiguous slab of MMA-N output rows). The candidates for a given CTA exist only after its MMA completes; candidates across CTAs exist only after all CTAs complete — i.e. after the full kernel-launch boundary.

**Outcome:** N's candidates are produced **before** `at::topk` runs (they all exist in `scores_all` when the next kernel starts). This is a **single-pass Gate**, not a Streaming Gate.

**Streaming Gate is NOT triggered by this delta.** Pre-wired decisions for Streaming Gate (file 08 section "Pre-Wired Decisions for d1.md") are not required. Output write is NOT deferred to the tile loop; accumulation buffer is NOT a SMEM top-K heap; threshold state is not maintained; Case A/Case B merge does not apply.

**Re-evaluation gate (Section 10, "Optimization re-evaluation rule"):** The dominant cost center after Tier 3 will need re-measurement. If post-Tier-3 `at::topk` is dominant AND K/N < 0.5 (true for `max_num_pages ≥ 65`, ~30% of the workload distribution), **a future Tier 2-class Streaming Gate optimization becomes the next candidate**. That is a separate optimization, not bundled into this delta (Section 15: "One optimization at a time").

## Atoms active in this delta

| Atom | Role in Tier 3 delta |
|---|---|
| FXP | **Primary, promoted to explicit.** Static iteration count = ceil(D=128 / K_slab=32) = 4. Online merge NO (MMA is a pure sum accumulator, identity = 0, written into TMEM via `enable_input_d=true` after slab 0). |
| PRD | **Primary, now hardware-delegated.** Tensor-core predicate is masked by zero-fill of K padding rows (tok ≥ seq_len) — the exact same trick Tier 2 uses, preserved here. |
| MOR | **Primary.** Dequant morphism (fp8 bytes + scale) → FP32 K_deq is replaced by a composed morphism (fp8 bytes → FP8 MMA operand + scale deferred → post-MMA column multiply). Composition is exact because scales are non-negative and `relu` commutes with non-negative scalar multiply; this identity is a new correctness claim that V1/V2 must verify. |
| MEA | Secondary. Four key measures change: (1) K tile byte count is now 128 FP8 bytes + 4 scale bytes per token (the raw layout, not the FP32-materialized 512 bytes); (2) reuse count of Q across N-tiles within one batch element is `ceil(N_max/MMA_N)`; (3) SMEM budget now includes Q and K FP8 tiles plus TMEM descriptors; (4) register budget includes MMA IDESC + SMEM descriptors. Numerical values bound in D2. |
| REL | Combine Group: MMA slab outputs across the 4 K slabs must merge into one TMEM accumulator per (M, N) cell. This is the MMA hardware's implicit Combine Group — `enable_input_d=true` after slab 0 enforces it. No cross-phase write→read hazards introduced beyond those mandated by GT-2's TMEM lifecycle. |
| SYM | Q along H=64 axis: AREA (each head scores independently; coalesced along H for TMA). K along N axis: AREA (each N position scores independently). D=128 head-dim: REDUCE (summed inside MMA). Bank symmetry handled by swizzle decision deferred to D3 (GT-10 constraint). |
| FUN | Grid→CTA functor: each CTA owns one (batch, N-tile) pair. CTA→Warp functor: 4 warps cooperate on one MMA instance (M=64 split into 4 warps × 16 rows per GT-12). Warp→Thread: tcgen05.ld distributes M-rows across lanes per GT-12. |
| SRG | Unchanged (H-reduction in post-MMA scoring kernel). |
| MON | Unchanged (top-K rank inside `at::topk`). |
| ATO | Inactive. |
| AFM | Affine maps for global K gather: `addr(phys, tok, k) = k_cache_base + phys*PAGE_BYTES + tok*128 + k`; scale: `scale_addr(phys, tok) = k_cache_base + phys*PAGE_BYTES + PAGE_DATA_BYTES + tok*4`. These are the same Tier 2 maps, not new. SMEM tile fill map into 8×T core-tile layout deferred to D3 (GT-11). |

## Pre-wired decisions for this delta (must survive to V1/V2/D3)

1. **Scale-application position — pre-relu, post-MMA, inside the fused kernel.** The correctness identity is the plain distributive law: `Σ_k Q * (fp8_to_fp32(K) * scale_tok) = scale_tok * Σ_k Q * fp8_to_fp32(K)`. This holds for any real `scale_tok` without sign assumptions. The fused kernel computes the inner sum via FP8 MMA into TMEM, then multiplies each output column by its `scale_tok` (gathered via `block_table[b, page_id]` → per-page scale vector) before writing `scores_all`. Downstream `weighted_relu_sum_batched_kernel` sees a bit-equivalent tensor to Tier 2's `torch::bmm(q_fp32, K_deq.T)` output and therefore preserves Tier 2's correctness chain exactly. V2 must verify that the scale is gathered from the SAME page-table entry used to gather the FP8 payload for each column (no index drift).
2. **Padding mask via zero K rows:** Preserved from Tier 2 GT-26. Rows with `tok ≥ seq_lens[b]` have their FP8 K payload forced to zero **before MMA**; zero FP8 bytes decode to 0.0 so MMA produces 0 at those columns; the scoring kernel then stamps `-INF` via the existing `seq_lens` check. GT-26 correctness chain preserved.
3. **MMA tile shape deferred to D3** (Table 41 lookup required); candidates are M=64 (matches H=64 exactly), N ∈ {64, 128, 256} (bound in D3), K=32 (mandatory for kind::f8f6f4 per slab).
4. **K loads via cooperative ld.global (GT-9):** TMA forbidden because page stride = 132 is not a multiple of 16. Cooperative SMEM fill of the 8×T core-tile layout (GT-11) required. Per-token scale bytes gathered into an SMEM scale vector in parallel with K bytes.
5. **Q loads:** Evaluate TMA vs cooperative in D3. Q has stride 128 × 1 = 128 bytes/row (multiple of 16), so TMA is permitted. Small enough (B×8192 bytes) that cooperative may also suffice. Bound in D3.
6. **TMEM lifecycle:** Per GT-2 — `tcgen05.alloc` by warp 1, `elect_one_sync` issues MMA, `tcgen05.commit` + `mbarrier.try_wait` + `tcgen05.fence::after_thread_sync` + `tcgen05.ld` + `tcgen05.wait::ld` + `tcgen05.dealloc`.
7. **Streaming Gate explicitly NOT pre-wired** (see detection result above).
